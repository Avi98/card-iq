# CardIQ

Expo SDK 54 / React Native 0.81.5 project. TypeScript codebase with strict mode enabled.

## Dev commands

```bash
npm start          # Expo dev server
npm run android    # Build Android app
npm run ios        # Build iOS app
```

## Key facts

- **Entry point**: `index.js` → `app/_layout.tsx` (expo-router)
- **Routes**: `app/` directory (expo-router file-based routing)
- **Screens**: `src/screens/` or `app/` directories
- **JS engine**: Hermes (enabled)
- **New Architecture**: Enabled (Fabric + TurboModules)
- **iOS deployment target**: 15.1+
- **No tests**: No Jest or test framework configured
- **No linting**: No ESLint configured
- **TypeScript**: Enabled with strict mode (`tsconfig.json`)

## Expo Router

`expo-router` is listed as a plugin in `app.json`. The app entry point references `.expo/.virtual-metro-entry` (Expo's virtual module entry for dev client and router). If adding routes, use `app/` directory convention per expo-router docs.

## Deep linking

- iOS URL scheme: `com.anonymous.CardIQ`
- iOS bundle ID: `com.anonymous.CardIQ`
- Android package: `com.anonymous.CardIQ`

These use `anonymous` placeholder — run `npx expo prebuild` to regenerate native projects with a real package name.

## Native projects

- `ios/` and `android/` are generated by Expo (managed workflow). Do not edit manually; run `npx expo prebuild` to regenerate after config changes.
- iOS uses Swift AppDelegate (`ios/CardIQ/AppDelegate.swift`)
- Android uses Kotlin (`android/app/src/main/java/com/anonymous/CardIQ/`)

## Dependencies

Only Expo modules and React Native. Key packages:
- `expo-router` (plugin)
- `expo-secure-store` (plugin)
- `expo-web-browser` (plugin)
- `expo-auth-session` (plugin)
